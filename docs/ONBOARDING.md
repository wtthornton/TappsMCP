# Getting Started with tapps-mcp

## Prerequisites

- **Python 3.12+**
- **[uv](https://docs.astral.sh/uv/)** (recommended) or pip
- Git

Optional for full pipeline: **tapps-brain** HTTP service at `localhost:8080` (see [operations/TAPPS-BRAIN-LOCAL-SETUP.md](operations/TAPPS-BRAIN-LOCAL-SETUP.md))

## Installation

From a consuming project (after global install):

```bash
uv tool install --from /path/to/tapps-mcp tapps-mcp
tapps-mcp init --host cursor
```

For **this monorepo** (development):

```bash
git clone https://github.com/wtthornton/tapps-mcp.git
cd tapps-mcp
uv sync --all-packages
```

Install global CLIs from the workspace (required for Cursor NLT serve scripts):

```bash
uv tool install --reinstall --from packages/tapps-mcp tapps-mcp
uv tool install --reinstall --from packages/docs-mcp docs-mcp
```

## Project Structure

```
tapps-mcp/
├── packages/
│   ├── tapps-core/     # Shared config, security, brain bridge, knowledge
│   ├── tapps-mcp/      # Code quality MCP server (32 tools)
│   └── docs-mcp/       # Documentation MCP server
├── docs/               # Human-facing documentation (start at INDEX.md)
├── .cursor/            # Cursor hooks, agents, skills (generated by tapps_init)
├── AGENTS.md           # Agent tool reference
└── .tapps-mcp.yaml     # Project configuration
```

## MCP Session Modes (NLT)

Enable **1–3** servers per session — not all six. Default after `tapps_init` is the **developer** bundle.

| Mode | Servers | When |
|------|---------|------|
| **Developer (default)** | `nlt-build`, `nlt-memory`, `nlt-linear-issues` | Daily coding + recall + backlog |
| **Build only** | `nlt-build` | Token-tight sessions (`--bundle minimal`) |
| **Build + Docs** | `nlt-build`, `nlt-project-docs` | Documentation generation / drift audit |
| **Setup** | `nlt-setup` | Bootstrap, upgrade, doctor only |

Details: [ADR-0016](adr/0016-needs-based-nlt-mcp-taxonomy.md).

## Key Concepts

- **TappsMCP** — Deterministic scoring (ruff, mypy, bandit, radon, vulture), quality gates, security scan, docs lookup
- **DocsMCP** — Doc generation, link/drift/completeness checks, diagrams, ADRs
- **TAPPS pipeline** — `tapps_session_start` → edit loop with `tapps_quick_check` → `tapps_validate_changed` → `tapps_checklist`
- **tapps-brain** — Cross-session memory (HTTP bridge); configured in `.tapps-mcp.yaml`

## Running the Servers

```bash
# TappsMCP (stdio)
uv run tapps-mcp serve --profile nlt-build

# DocsMCP (stdio)
uv run docsmcp serve

# Diagnostics
uv run tapps-mcp doctor --quick
```

## Running Tests

```bash
uv run pytest packages/docs-mcp/tests/ -m "not slow" -v
```
