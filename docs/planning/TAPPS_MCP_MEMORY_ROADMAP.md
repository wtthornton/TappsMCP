## Overview

TappsMCP currently exposes ~40% of tapps-brain's capabilities through 23 `tapps_memory` actions. This roadmap closes the feature gap so that **code projects never need to run `tapps-brain-mcp` alongside TappsMCP**. It adds 15 new actions, wires memory into the quality pipeline, enables Hive for Agent Teams, and surfaces OWASP ASI06 security defenses.

All changes are scoped to `packages/tapps-mcp/` and `packages/tapps-core/`. Changes needed in the tapps-brain repository are listed as **prerequisites** (not detailed here -- see the companion doc).

## Architecture Principle

**TappsMCP is the single gateway for code projects. tapps-brain is the engine underneath.**

```
MCP Client (Claude Code, Cursor, etc.)
        |
        v
  tapps_memory (single MCP tool, expanding from 23 to 38 actions)
        |
        v
  server_memory_tools.py (dispatch + response curation)
        |
        v
  tapps_core.memory.* (re-export shims)
        |--- injection.py (bridge: TappsMCP settings --> tapps-brain InjectionConfig)
        +--- *.py (thin re-exports)
        |
        v
  tapps_brain (standalone library)
```

**Rules:**
1. Never run `tapps-brain-mcp` alongside TappsMCP on the same project
2. TappsMCP adds value over raw tapps-brain through pipeline integration, profile auto-detection, quality-informed memory, and Agent Teams coordination
3. Hive (`~/.tapps-brain/hive/`) and Federation (`~/.tapps-brain/memory/federated.db`) are cross-project infrastructure safe to share across gateways
4. All new actions flow through the existing `tapps_memory` tool -- no new MCP tool registrations

## Epic M1: Security Surface

**Priority:** P0
**Effort:** 2-3 days
**Motivation:** OWASP ASI06 (Memory & Context Poisoning) is now a Top 10 item for agentic applications. Microsoft documented 50 real-world memory poisoning attacks. TappsMCP must surface tapps-brain's safety defenses and fill gaps.

These must land in tapps-brain first:
- R01: `check_content()` enforced on every `MemoryStore.save()` by default
- R02: `integrity_hash` column + `verify_integrity()` method
- R03: Sliding-window rate limiter on writes with anomaly logging

```python
def _auto_detect_memory_profile(project_info: dict) -> str:
    """Select best memory profile based on project type."""
    # Priority 1: explicit setting in .tapps-mcp.yaml
    if settings.memory.profile:
        return settings.memory.profile

    # Priority 2: project-level .tapps-brain/profile.yaml
    profile_yaml = project_root / ".tapps-brain" / "profile.yaml"
    if profile_yaml.exists():
        return "project_override"

    # Priority 3: auto-detect from project type
    project_type = project_info.get("project_type", "")
    tech_stack = project_info.get("tech_stack", [])

    if any(t in tech_stack for t in ["jupyter", "pandas", "numpy", "scikit-learn", "tensorflow", "pytorch"]):
        return "research-knowledge"
    if project_info.get("is_monorepo") and any(t in tech_stack for t in ["github-actions", "gitlab-ci"]):
        return "project-management"

    return "repo-brain"  # default
```

## Epic M3: Hive & Agent Teams

**Priority:** P1
**Effort:** 3-4 days
**Motivation:** Claude Code Agent Teams (`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`) have no shared memory. tapps-brain's Hive provides namespace-isolated multi-agent memory. This is TappsMCP's differentiator for multi-agent workflows.

### M3.1: HiveStore Singleton Management

**File:** `server_helpers.py`

Add alongside existing `_get_memory_store()`:

```python
_hive_store: HiveStore | None = None
_hive_lock = threading.Lock()

def _get_hive_store() -> HiveStore | None:
    """Return HiveStore singleton, or None if Agent Teams not enabled."""
    global _hive_store
    if not os.environ.get("CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS"):
        return None
    with _hive_lock:
        if _hive_store is None:
            from tapps_brain.hive import HiveStore
            _hive_store = HiveStore()
        return _hive_store

def _reset_hive_store_cache() -> None:
    global _hive_store
    _hive_store = None
```

```python
async def _auto_save_expert_result(domain: str, question: str, answer: str) -> None:
    """Save expert consultation result to memory if enabled."""
    settings = load_settings()
    if not settings.memory.auto_save_quality:

    store = _get_memory_store()
```

### M4.2: Auto-Save Recurring Quality Issues

**File:** `server_helpers.py`

Track recurring failures across `tapps_quick_check` calls. When the same file fails the same category 3+ times in the store's metrics, save a procedural memory:

```python
async def _track_recurring_failures(file_path: str, failures: list[str]) -> None:
    """Save procedural memory when a file repeatedly fails the same category."""
    store = _get_memory_store()
    if not store:
        return

    for category in failures:
        key = f"recurring-{_slugify(file_path)}-{category}"
        existing = store.get(key)
        if existing:
            store.reinforce(key, confidence_boost=0.1)
        else:
            store.save(
                key=key,
                value=f"File {file_path} has recurring {category} issues. Review and address root cause.",
                tier="procedural",
                source="agent",
                scope="project",
                tags=["recurring-issue", category, "auto-captured"],
            )
```

### M4.3: Auto-Supersede for Architectural Saves

**File:** `server_memory_tools.py`

In the `save` action handler, when `tier="architectural"` and the key already exists, use `supersede()` instead of overwrite:

```python
if action == "save":
    tier = kwargs.get("tier", "context")
    key = kwargs.get("key", "")

    if tier == "architectural" and key:
        existing = store.get(key)
        if existing and existing.tier == "architectural":
            # Use bi-temporal supersede to preserve history
            result = store.supersede(old_key=key, new_value=value, tier=tier, ...)
```

```python
def _enrich_impact_with_memory(file_path: str) -> list[dict]:
    """Query memory relations for semantic connections to this file."""
    store = _get_memory_store()
    if not store:
        return []

    # Search for memories mentioning this file
    results = store.search(file_path, limit=5, min_confidence=0.3)

    # Also check relation graph
    relations = store.find_related(file_path)

    context = []
    for entry in results:
        context.append({
            "key": entry.key,
            "summary": entry.value[:100],
            "tier": entry.tier,
            "source": "memory",
        })
    for rel in relations:
        context.append({
            "subject": rel.subject,
            "predicate": rel.predicate,
            "object": rel.object_entity,
            "source": "relation_graph",
        })
    return context
```
