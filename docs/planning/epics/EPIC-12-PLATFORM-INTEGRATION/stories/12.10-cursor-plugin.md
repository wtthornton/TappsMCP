**Status:** Complete
**Tier:** 3

## Summary

Cursor supports a plugin format that bundles MCP configuration, skills, subagent
definitions, hooks, and rules into a single installable directory structure rooted
at `.cursor-plugin/`. Publishing TappsMCP as a Cursor plugin allows users to
install all quality tooling in one step via `cursor://install-plugin/tapps-mcp-plugin`
or through the Cursor marketplace, rather than running `tapps_init` and manually
configuring each component. This story creates the `.cursor-plugin/` directory
structure inside the TappsMCP repository so it can be distributed alongside the
Python package, bundles the outputs from stories 12.7 (Cursor hooks), 12.8 (skills),
and the Cursor-specific subagent definitions into the plugin, and adds a `plugin.json`
manifest with all required fields including `name`, `displayName`, `author`,
`description`, `keywords`, `license`, and `version`.

```
    tapps-python-quality.mdc
    tapps-expert-consultation.mdc

```

### plugin.json format

```json
{
  "name": "tapps-mcp-plugin",
  "displayName": "TappsMCP Quality Tools",
  "author": "TappsMCP Team",
  "description": "Code quality scoring, security scanning, and quality gates for Python projects",
  "keywords": ["code-quality", "security", "scoring", "mcp", "python"],
  "license": "MIT",
  "version": "1.0.0"
}
```

All seven fields are required by the Cursor plugin spec. The `name` field must
be lowercase kebab-case. The `version` field must be kept in sync with
`pyproject.toml`; consider reading it dynamically or generating the file as
part of the build process.

### mcp.json format

The plugin-level `mcp.json` is distinct from the project-level `.cursor/mcp.json`
and uses the same schema:

```json
{
  "mcpServers": {
    "tapps-mcp": {
      "command": "uvx",
      "args": ["tapps-mcp", "serve"],
      "env": {
        "TAPPS_MCP_PROJECT_ROOT": "${workspaceFolder}"
      }
    }
  }
}
```

### Cursor subagent format differences

Cursor subagents use YAML arrays for `tools` (not comma-separated strings like
Claude Code) and support additional fields: `readonly`, `is_background`, and
`model: inherit`. The plugin agents directory must contain Cursor-format files,
not copies of the Claude Code format agents. Create a separate set of agent
templates in `src/tapps_mcp/prompts/agents/cursor/` to serve as the source of
truth for the Cursor plugin.

### Approach

1. Create the `plugin/cursor/` directory tree in the repository.
2. Store `plugin.json` as a static file; add a CI check that verifies
   the version matches `pyproject.toml`.
3. Keep Cursor agent templates separate from Claude Code agent templates in
   `src/tapps_mcp/prompts/agents/cursor/` to handle the format differences.
4. The `hooks/hooks.json` in the plugin uses the 6 Cursor hook events
   (`beforeSubmitPrompt`, `beforeShellExecution`, `beforeMCPExecution`,
   `beforeReadFile`, `afterFileEdit`, `stop`).
5. Include the entire `plugin/` directory in `MANIFEST.in` and
   `pyproject.toml` `[tool.setuptools.package-data]` so it is distributed
   with the package.
