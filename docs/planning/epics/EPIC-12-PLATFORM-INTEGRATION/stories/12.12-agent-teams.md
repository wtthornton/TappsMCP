# Story 12.12: Agent Teams Integration

**Status:** Pending
**Tier:** 3
**Priority:** P2
**LOE:** 1 day
**Platform:** Claude Code

---

## Summary

Claude Code's Agent Teams feature allows multiple independent Claude Code
instances to collaborate on a shared task list, with one acting as team lead
and others as teammates. This is distinct from subagents (which run within a
single Claude Code session's context) — teammates are fully separate processes
with their own context windows that communicate via a JSON mailbox file system.
TappsMCP can act as a "quality watchdog" teammate that monitors file changes,
runs quality checks on modified Python files, and prevents tasks from being
marked complete until quality gates pass. This story adds Agent Teams guidance
to the `CLAUDE.md` generated by `tapps_init`, documents the "quality watchdog"
pattern, and adds two team-specific hook configurations: a `TeammateIdle` hook
that uses exit 2 to keep the quality watchdog active, and a `TaskCompleted`
hook that uses exit 2 to prevent task completion if quality gates fail. Both
hooks are added as optional blocks in the generated `.claude/settings.json`
behind an Agent Teams configuration flag so they do not affect projects that
are not using Agent Teams.

## Acceptance Criteria

- The `CLAUDE.md` generated by `tapps_init` includes an "Agent Teams" section
  that documents the quality watchdog pattern
- The Agent Teams section in `CLAUDE.md` mentions the
  `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` environment variable required to
  enable the feature
- The Agent Teams section describes designating one teammate as the quality
  watchdog and its responsibilities (monitor changes, run `tapps_quick_check`,
  message teammates about issues)
- The Agent Teams section explains that the `TaskCompleted` hook prevents tasks
  from being marked complete if quality gates fail
- When `tapps_init` is called with `agent_teams: true` (or an equivalent
  opt-in parameter), the generated `.claude/settings.json` includes a
  `TeammateIdle` hook configuration
- The `TeammateIdle` hook script uses exit 2 with stderr output to keep the
  quality watchdog teammate active when quality checks are pending
- When `tapps_init` is called with `agent_teams: true`, the generated
  `.claude/settings.json` includes a `TaskCompleted` hook configuration
- The `TaskCompleted` hook script calls `tapps_validate_changed` (or equivalent
  quality gate check) and exits 2 with a descriptive stderr message if the gate
  fails
- Both team hooks are omitted from `.claude/settings.json` when Agent Teams is
  not opted in (default)
- Hook scripts for Agent Teams (`tapps-teammate-idle.sh`, `tapps-task-completed.sh`)
  are generated in `.claude/hooks/`
- A unit test verifies that Agent Teams hooks are present in `settings.json`
  when the opt-in flag is set
- A unit test verifies that Agent Teams hooks are absent when the opt-in flag
  is not set
- `mypy --strict` passes with no new errors
- `ruff check` passes with no new errors

### TeammateIdle hook script

```bash
#!/usr/bin/env bash
# tapps-teammate-idle.sh — Keep quality watchdog active while gates are pending
set -euo pipefail

INPUT=$(cat)
TEAMMATE=$(echo "$INPUT" | jq -r '.teammate_name // ""')

# Only keep the quality watchdog active
if [[ "$TEAMMATE" != "tapps-quality-watchdog" ]]; then
  exit 0
fi

# Check if any files still fail quality gates
RESULT=$(mcp__tapps-mcp__tapps_validate_changed 2>/dev/null || echo "pending")
if echo "$RESULT" | grep -q '"passed": false'; then
  echo "Quality gates not yet passing — continue monitoring" >&2
  exit 2
fi

exit 0
```

### TaskCompleted hook script

```bash
#!/usr/bin/env bash
# tapps-task-completed.sh — Prevent task completion if quality gates fail
set -euo pipefail

INPUT=$(cat)

# Run quality gate check
RESULT=$(mcp__tapps-mcp__tapps_validate_changed 2>/dev/null || true)
if echo "$RESULT" | grep -q '"passed": false'; then
  echo "Task blocked: quality gates not passing. Run tapps_validate_changed for details." >&2
  exit 2
fi

exit 0
```

### Agent Teams section in CLAUDE.md template

```markdown
## Agent Teams (Optional)

If using Claude Code Agent Teams (`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`),
designate one teammate as a **quality watchdog**:

1. The quality watchdog runs `tapps_quick_check` on files changed by other
   teammates.
2. It messages other teammates via the shared mailbox when quality issues are
   found.
3. The `TaskCompleted` hook prevents any task from being marked complete until
   `tapps_validate_changed` passes.
4. The `TeammateIdle` hook keeps the watchdog active while quality issues
   remain unresolved.

To enable Agent Teams hooks, re-run `tapps_init` with `agent_teams=True`.
```

### settings.json TeammateIdle hook block

```json
"TeammateIdle": [
  {
    "matcher": "tapps-quality-watchdog",
    "hooks": [
      {
        "type": "command",
        "command": ".claude/hooks/tapps-teammate-idle.sh"
      }
    ]
  }
]
```

### settings.json TaskCompleted hook block

```json
"TaskCompleted": [
  {
    "hooks": [
      {
        "type": "command",
        "command": ".claude/hooks/tapps-task-completed.sh"
      }
    ]
  }
]
```

### Approach

1. Add an `agent_teams: bool = False` parameter to the `tapps_init` MCP tool
   handler in `server_pipeline_tools.py`.
2. Pass the parameter through to `pipeline/init.py` and
   `distribution/setup_generator.py`.
3. In `setup_generator.py`, when `agent_teams=True`, append the
   `TeammateIdle` and `TaskCompleted` hook blocks to the `hooks` section of
   the generated `.claude/settings.json`.
4. Generate the two hook scripts regardless of the `agent_teams` flag so
   they are available if a user enables Agent Teams later.
5. Add the Agent Teams section to the `CLAUDE.md` template unconditionally
   (it is documentation, not configuration, so it is safe to include always).
