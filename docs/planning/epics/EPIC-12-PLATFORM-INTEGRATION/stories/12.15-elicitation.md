# Story 12.15: MCP Elicitation Support

**Status:** Complete
**Tier:** 4
**Priority:** P3
**LOE:** 2-3 days
**Platform:** Cursor

---

## Summary

MCP elicitation is a protocol extension that allows MCP servers to request
structured input from the user via the host's native UI during tool execution.
Cursor is currently the only MCP client that implements elicitation. When a
TappsMCP tool sends an `elicitation/create` request, Cursor displays a native
dialog with form fields defined by a JSON Schema, and returns the user's
selection to the tool before execution continues. This story adds elicitation
to two TappsMCP tool handlers: `tapps_quality_gate` (to ask which preset to
use when none is specified) and `tapps_init` (to confirm the operation before
writing configuration files). Elicitation uses string enums with `enumNames`
for display labels, giving Cursor users an interactive, frictionless way to
select presets without needing to know the exact preset name string. On
non-Cursor clients that do not support elicitation, the tool falls back to
its existing default behavior.

## Acceptance Criteria

- `tapps_quality_gate` sends an elicitation request when the `preset` parameter
  is not provided by the caller
- The elicitation request for `tapps_quality_gate` uses a string enum with
  values `["development", "staging", "production"]` and `enumNames`
  `["Development (lenient)", "Staging (moderate)", "Production (strict)"]`
- When the user selects a preset via elicitation, `tapps_quality_gate` uses
  that preset value to run the gate
- When the user declines or cancels elicitation, `tapps_quality_gate` returns
  an informative message rather than raising an error
- `tapps_init` sends an elicitation request to confirm the operation before
  writing any files, when the client supports elicitation
- The elicitation request for `tapps_init` presents a boolean confirmation
  with a message describing what files will be written
- When the user declines or cancels `tapps_init` elicitation, no files are
  written and an informative message is returned
- Elicitation is only attempted when the MCP session supports the
  `elicitation` capability; on unsupported clients the tools fall back to
  existing behavior (use default preset for `tapps_quality_gate`, proceed
  without confirmation for `tapps_init`)
- The elicitation capability check is done against the MCP session context,
  not by hardcoding a client name
- A unit test verifies the elicitation request schema structure for
  `tapps_quality_gate`
- A unit test verifies the elicitation request schema structure for
  `tapps_init`
- A unit test verifies fallback behavior when elicitation is not supported
- `mypy --strict` passes with no new errors
- `ruff check` passes with no new errors

### Elicitation request format

```python
async def request_elicitation(
    session: Any,
    message: str,
    schema: dict[str, Any],
) -> dict[str, Any] | None:
    """
    Send an elicitation/create request to the MCP client.
    Returns the content dict if accepted, None if declined or unsupported.
    """
    try:
        response = await session.send_request(
            "elicitation/create",
            {
                "message": message,
                "requestedSchema": schema,
            },
        )
        if response.get("action") == "accept":
            return response.get("content", {})
        return None
    except Exception:
        # Client does not support elicitation — fall back silently
        return None
```

### Quality gate preset elicitation schema

```python
PRESET_ELICITATION_SCHEMA = {
    "type": "object",
    "properties": {
        "preset": {
            "type": "string",
            "enum": ["development", "staging", "production"],
            "enumNames": [
                "Development (lenient — score >= 60)",
                "Staging (moderate — score >= 75)",
                "Production (strict — score >= 90)",
            ],
        }
    },
    "required": ["preset"],
}
```

### tapps_quality_gate elicitation flow

```python
async def tapps_quality_gate(preset: str | None = None, ...) -> str:
    if preset is None:
        result = await request_elicitation(
            session,
            message="Which quality gate preset should be applied?",
            schema=PRESET_ELICITATION_SCHEMA,
        )
        if result is not None:
            preset = result["preset"]
        else:
            preset = "development"  # fallback default
    # ... existing gate logic
```

### tapps_init elicitation flow

```python
async def tapps_init(project_root: str, ...) -> str:
    if session_supports_elicitation(session):
        result = await request_elicitation(
            session,
            message=(
                f"TappsMCP will write configuration files to {project_root}. "
                "This includes .claude/settings.json, .mcp.json, CLAUDE.md, "
                "and AGENTS.md. Proceed?"
            ),
            schema=INIT_CONFIRMATION_SCHEMA,
        )
        if result is None or not result.get("confirm", False):
            return "tapps_init cancelled — no files were written."
    # ... existing init logic
```

### Capability detection

The MCP SDK exposes session capabilities on the server context object. Check
for elicitation support before attempting the request:

```python
def session_supports_elicitation(session: Any) -> bool:
    """Return True if the current MCP client supports elicitation."""
    try:
        caps = session.client_capabilities or {}
        return bool(caps.get("elicitation"))
    except AttributeError:
        return False
```

The exact attribute path depends on the `mcp` Python SDK version in use.
Verify against the installed version in `pyproject.toml` before implementing.

### Approach

1. Create `src/tapps_mcp/common/elicitation.py` with `request_elicitation`
   and `session_supports_elicitation` helpers.
2. Access the MCP session object from the tool handler context — the `mcp`
   Python SDK provides this via the `Context` or request context parameter.
   Check the SDK version for the correct API.
3. Modify `tapps_quality_gate` to call `request_elicitation` when `preset` is
   `None`.
4. Modify `tapps_init` to call `request_elicitation` for confirmation when
   elicitation is supported.
5. All elicitation calls are wrapped in `try/except` so that unsupported
   clients degrade gracefully without raising.

### Implementation risk

The `mcp` Python SDK's elicitation API may differ from the protocol spec.
Check `pyproject.toml` for the pinned `mcp` SDK version and review the SDK's
changelog or source for `elicitation/create` support before implementing.
If the SDK does not yet support elicitation as a first-class API, implement
it by directly calling `session.send_request("elicitation/create", ...)` on
the raw session object.
