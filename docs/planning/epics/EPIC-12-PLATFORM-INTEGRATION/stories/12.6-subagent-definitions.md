# Story 12.6: Custom Subagent Definitions

**Status:** Complete
**Tier:** 2
**Priority:** P1
**LOE:** 2-3 days
**Platform:** Claude Code + Cursor

---

## Summary

Both Claude Code and Cursor support custom subagent definitions — Markdown files
with YAML frontmatter that describe specialized AI assistants with their own tool
access, model selection, and system prompts. Claude Code subagents are placed in
`.claude/agents/` and Cursor subagents in `.cursor/agents/`. The two platforms
use different frontmatter schemas: Claude Code uses a comma-separated string for
`tools` while Cursor uses a YAML array. Currently `tapps_init` does not generate
any subagent definitions, meaning the quality-focused agents (reviewer, researcher,
validator) must be created manually by each team. This story adds subagent
definition generation to `tapps_init` for both platforms, producing three
ready-to-use subagent files per platform that integrate directly with the TappsMCP
MCP tools available in the session.

---

## Research Reference

See `../research/claude-code-subagents.md` for the complete frontmatter field
reference for both platforms, the per-platform format differences (comma-separated
vs YAML array tools), MCP tool access rules, key constraints (no nesting for
Claude Code, up to 4 concurrent for Cursor), and the full system prompt content
for all three TappsMCP subagents.

## Acceptance Criteria

- `tapps_init` generates the following files for Claude Code projects:
  - `.claude/agents/tapps-reviewer.md`
  - `.claude/agents/tapps-researcher.md`
  - `.claude/agents/tapps-validator.md`
- `tapps_init` generates the following files for Cursor projects (or when
  `--platform cursor` is specified):
  - `.cursor/agents/tapps-reviewer.md`
  - `.cursor/agents/tapps-researcher.md`
  - `.cursor/agents/tapps-validator.md`
- Claude Code agent files use a comma-separated string for the `tools` field
  (e.g., `tools: Read, Glob, Grep`)
- Cursor agent files use a YAML array for the `tools` field (e.g., `tools:\n  - code_search`)
- `tapps-reviewer` on both platforms has `model: sonnet` and `permissionMode: dontAsk`
- `tapps-researcher` on both platforms has `model: haiku`
- `tapps-validator` on both platforms has `model: sonnet` and `permissionMode: dontAsk`
- Claude Code `tapps-reviewer` includes `memory: project` in frontmatter
- Claude Code `tapps-researcher` includes `memory: project` in frontmatter
- All generated agent files include a `name` and `description` frontmatter field
- The system prompt body of each agent file references the correct MCP tool names
  using the `mcp__tapps-mcp__<tool_name>` pattern for Claude Code
- If an agent file already exists at the target path, `tapps_init` skips writing
  it and logs a warning rather than overwriting user customizations
- A unit test verifies that agent files are created with the correct content for
  each platform
- A unit test verifies the frontmatter `tools` field format differs correctly
  between Claude Code and Cursor
- `mypy --strict` passes with no new errors
- `ruff check` passes with no new errors

### Claude Code agent format

```markdown
---
name: tapps-reviewer
description: Use proactively to review code quality, run security scans, and enforce quality gates after editing Python files.
tools: Read, Glob, Grep
model: sonnet
permissionMode: dontAsk
memory: project
---

You are a TappsMCP quality reviewer. When invoked:

1. Identify which Python files were recently edited
2. Call `mcp__tapps-mcp__tapps_quick_check` on each changed file
3. If any file scores below 70, call `mcp__tapps-mcp__tapps_score_file` for detailed breakdown
4. Summarize findings: file, score, top issues, suggested fixes
5. If overall quality is poor, recommend calling `tapps_quality_gate`

Focus on actionable feedback. Be concise.
```

### Cursor agent format

Cursor uses a YAML array for `tools` instead of a comma-separated string. The
`permissionMode` field does not exist in Cursor's schema; use `readonly: false`
for agents that may need to act on findings. The `memory` field is not supported
in Cursor.

```markdown
---
name: tapps-reviewer
description: Use proactively to review code quality, run security scans, and enforce quality gates after editing Python files.
model: sonnet
readonly: false
is_background: false
tools:
  - code_search
  - read_file
---

You are a TappsMCP quality reviewer. When invoked:

1. Identify which Python files were recently edited
2. Call the `tapps_quick_check` MCP tool on each changed file
3. If any file scores below 70, call `tapps_score_file` for detailed breakdown
4. Summarize findings: file, score, top issues, suggested fixes
5. If overall quality is poor, recommend calling `tapps_quality_gate`

Focus on actionable feedback. Be concise.
```

### Agent content summary

| Agent | Model | Purpose | Key MCP tools |
|-------|-------|---------|----------------|
| `tapps-reviewer` | sonnet | Proactive quality checking after edits | `tapps_quick_check`, `tapps_score_file`, `tapps_quality_gate` |
| `tapps-researcher` | haiku | Documentation lookup and expert consultation | `tapps_research`, `tapps_consult_expert` |
| `tapps-validator` | sonnet | Pre-completion validation, reports only | `tapps_validate_changed` |

### Approach

1. In `pipeline/init.py`, add
   `_generate_subagent_definitions(project_root: Path, platform: str) -> None`.
2. Determine target directory: `.claude/agents/` for `claude` platform,
   `.cursor/agents/` for `cursor` platform.
3. Create the directory if it does not exist.
4. For each of the three agents, check if the target file exists. If it does,
   log a warning and skip. If not, read the appropriate template from
   `src/tapps_mcp/prompts/agents/{platform}/` and write it.
5. Include both `claude` and `cursor` template subdirectories in package data
   in `pyproject.toml`.
6. Call `_generate_subagent_definitions()` from the main init handler after
   determining the platform from the `tapps_init` arguments.
