# Story 12.1: Add Tool Annotations

**Status:** Pending
**Tier:** 1
**Priority:** P0
**LOE:** 1 day
**Platform:** All (Claude Code, Cursor, VS Code, any MCP client)

---

## Summary

MCP 2025-03-26 added four behavioral hint fields to the tool registration spec:
`readOnlyHint`, `destructiveHint`, `idempotentHint`, and `openWorldHint`. These
annotations let MCP clients make UX decisions (auto-approve, skip confirmation
dialogs, enable automatic retry) without needing to understand each tool's
implementation. Currently all 21 TappsMCP tools are registered without any
annotations, causing clients to apply the most conservative defaults:
non-read-only, potentially destructive, non-idempotent. This produces unnecessary
permission prompts on every invocation. Adding accurate annotations eliminates
friction across all MCP clients with no changes to tool behavior.

---

## Research Reference

See `../research/tool-annotations.md` for the full annotation spec, client
behavior rules, and the complete per-tool audit of all 21 TappsMCP tools.

---

## Acceptance Criteria

- All 21 `@mcp.tool()` decorators carry an explicit `annotations` dict with all
  four hint fields (`readOnlyHint`, `destructiveHint`, `idempotentHint`,
  `openWorldHint`) set to accurate values
- 18 of 21 tools have `readOnlyHint=True` (all tools except `tapps_session_start`,
  `tapps_feedback`, and `tapps_init`)
- 0 of 21 tools have `destructiveHint=True` (none are destructive)
- 19 of 21 tools have `idempotentHint=True` (all except `tapps_feedback` and
  `tapps_research` / `tapps_lookup_docs` which depend on external state)
- 2 of 21 tools have `openWorldHint=True` (`tapps_research` and
  `tapps_lookup_docs`, which may call the Context7 external API)
- All remaining 19 tools have `openWorldHint=False`
- A unit test verifies the annotation values on every registered tool
- `mypy --strict` passes with no new errors
- `ruff check` passes with no new errors

## Implementation Notes

### Files to change

| File | Change |
|------|--------|
| `src/tapps_mcp/server.py` | Add `annotations={}` to all tool decorators registered directly in this file |
| `src/tapps_mcp/server_scoring_tools.py` | Add `annotations={}` to `tapps_score_file`, `tapps_quality_gate`, `tapps_quick_check` |
| `src/tapps_mcp/server_pipeline_tools.py` | Add `annotations={}` to `tapps_validate_changed`, `tapps_session_start`, `tapps_init` |
| `src/tapps_mcp/server_metrics_tools.py` | Add `annotations={}` to `tapps_dashboard`, `tapps_stats`, `tapps_feedback`, `tapps_research` |

### Annotation values per tool

The complete audit from `../research/tool-annotations.md` defines the following
values. Apply them exactly:

**Read-only, idempotent, closed-world (16 tools):**
`tapps_score_file`, `tapps_quality_gate`, `tapps_quick_check`,
`tapps_validate_changed`, `tapps_consult_expert`, `tapps_dashboard`,
`tapps_stats`, `tapps_checklist`, `tapps_validate_config`,
`tapps_impact_analysis`, `tapps_project_profile`, `tapps_session_notes`,
`tapps_adaptive_weights`, `tapps_explain_score`, `tapps_compare_scores`,
`tapps_trend_analysis`

```python
annotations={
    "readOnlyHint": True,
    "destructiveHint": False,
    "idempotentHint": True,
    "openWorldHint": False,
}
```

**Read-only, non-idempotent, open-world (2 tools):**
`tapps_research`, `tapps_lookup_docs`

```python
annotations={
    "readOnlyHint": True,
    "destructiveHint": False,
    "idempotentHint": False,
    "openWorldHint": True,
}
```

**Side-effect, idempotent, closed-world (2 tools):**
`tapps_session_start`, `tapps_init`

```python
annotations={
    "readOnlyHint": False,
    "destructiveHint": False,
    "idempotentHint": True,
    "openWorldHint": False,
}
```

**Side-effect, non-idempotent, closed-world (1 tool):**
`tapps_feedback`

```python
annotations={
    "readOnlyHint": False,
    "destructiveHint": False,
    "idempotentHint": False,
    "openWorldHint": False,
}
```

### Approach

1. Audit each `@mcp.tool()` call across all four server files to confirm which
   file each tool is defined in.
2. Add the `annotations` keyword argument to the decorator call. If the decorator
   currently takes only `name` or `description`, add `annotations` as an additional
   keyword argument — do not replace existing arguments.
3. Use the exact boolean values from the table above; do not guess.

## Testing Requirements

### Unit tests — `tests/unit/test_tool_annotations.py` (new file)

- Import the MCP server app object and iterate over registered tools.
- For each of the 21 tools, assert that `tool.annotations` is not `None`.
- Assert `readOnlyHint` is `True` for the 18 read-only tools.
- Assert `readOnlyHint` is `False` for `tapps_session_start`, `tapps_feedback`,
  `tapps_init`.
- Assert `destructiveHint` is `False` for all 21 tools.
- Assert `idempotentHint` is `True` for all tools except `tapps_feedback`.
- Assert `openWorldHint` is `True` only for `tapps_research` and
  `tapps_lookup_docs`.
- Assert `openWorldHint` is `False` for all other 19 tools.
