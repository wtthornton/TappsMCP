**Status:** Complete

## Summary

The Cursor plugin bundle created in story 12.10 can be submitted to the Cursor
marketplace at `cursor.com/marketplace`, enabling users to discover and install
TappsMCP in one click via the in-editor `/add-plugin` command or the deep link
`cursor://install-plugin/tapps-mcp-plugin`. This story prepares TappsMCP for
marketplace submission by creating a `marketplace.json` manifest with all
required metadata, adding a logo image, ensuring the README and CHANGELOG are
marketplace-ready, running the Cursor plugin validation script, and documenting
the submission process. Publishing to the marketplace significantly lowers the
barrier to adoption for Cursor users compared to manual configuration.

## Acceptance Criteria

- A `marketplace.json` file is present in the `plugin/cursor/` directory with
  all marketplace-required fields beyond the base `plugin.json`
- `marketplace.json` includes: `name`, `displayName`, `author`, `description`,
  `keywords`, `license`, `version`, `repository`, `homepage`, `category`
- The `category` field is set to a valid Cursor marketplace category
  (e.g., `"code-quality"` or `"developer-tools"`)
- The `repository` field points to the TappsMCP GitHub repository URL
- A `logo.png` file is present in `plugin/cursor/` at a minimum size of
  128x128 pixels
- The `plugin/cursor/README.md` includes:
  - Installation instructions (via marketplace, deep link, and `/add-plugin`)
  - A "Quick Start" section with the first 3 steps after installation
  - A "Requirements" section listing Python 3.12+ and `uvx`/`uv`
  - A screenshot or GIF placeholder path
- A `CHANGELOG.md` is present in `plugin/cursor/` documenting the initial
  release as version 1.0.0
- The Cursor plugin validation script (`node scripts/validate-template.mjs`)
  passes without errors when run against the `plugin/cursor/` directory
- A `scripts/validate-cursor-plugin.sh` helper script is added to the
  TappsMCP repository to run validation
- The `Makefile` or `pyproject.toml` `[tool.scripts]` section includes a
  `validate-cursor-plugin` target
- All validation passes in CI (add to GitHub Actions)
- A unit test verifies `marketplace.json` contains all required fields
- A unit test verifies `marketplace.json` is valid JSON
- `mypy --strict` passes with no new errors
- `ruff check` passes with no new errors

### scripts/validate-cursor-plugin.sh

```bash
#!/usr/bin/env bash
# Validate the Cursor plugin bundle structure and manifest
set -euo pipefail

PLUGIN_DIR="${1:-plugin/cursor}"

echo "Validating Cursor plugin bundle at: $PLUGIN_DIR"

# Check required files
REQUIRED_FILES=(
  ".cursor-plugin/plugin.json"
  "marketplace.json"
  "mcp.json"
  "logo.png"
  "README.md"
  "CHANGELOG.md"
  "skills/tapps-score/SKILL.md"
  "skills/tapps-gate/SKILL.md"
  "skills/tapps-validate/SKILL.md"
)

for f in "${REQUIRED_FILES[@]}"; do
  if [[ ! -f "$PLUGIN_DIR/$f" ]]; then
    echo "ERROR: Missing required file: $PLUGIN_DIR/$f" >&2
    exit 1
  fi
done

# Validate plugin.json is valid JSON with all required fields
python3 -c "
import json, sys
with open('$PLUGIN_DIR/.cursor-plugin/plugin.json') as f:
    p = json.load(f)
required = ['name', 'displayName', 'author', 'description', 'keywords', 'license', 'version']
missing = [k for k in required if k not in p]
if missing:
    print(f'ERROR: plugin.json missing fields: {missing}', file=sys.stderr)
    sys.exit(1)
print('plugin.json: OK')
"

echo "Plugin validation passed."
```

### Approach

1. Create `marketplace.json` as a static file in `plugin/cursor/`.
2. Design or source a 128x128 PNG logo and place it at `plugin/cursor/logo.png`.
3. Update `plugin/cursor/README.md` with marketplace-specific installation
   instructions and quick start.
4. Create `plugin/cursor/CHANGELOG.md` with the initial release entry.
5. Add `scripts/validate-cursor-plugin.sh` as an executable shell script.
6. Add a `.github/workflows/validate-cursor-plugin.yml` that runs the validation
   script in CI on pull requests that touch `plugin/cursor/`.
7. Document the submission process in `docs/cursor-marketplace-publishing.md`
   with step-by-step instructions for publishing at `cursor.com/marketplace/publish`.

### Marketplace submission checklist

- `plugin.json` and `marketplace.json` must have matching `version` fields
- `version` must match `pyproject.toml`
- `logo.png` must be at least 128x128 pixels
- `README.md` must not contain broken links
- All `.mdc` rule files must have valid frontmatter
- Run `node scripts/validate-template.mjs plugin/cursor` before submitting
