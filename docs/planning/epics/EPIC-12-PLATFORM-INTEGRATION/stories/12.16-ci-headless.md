## Summary

Claude Code can run in headless (non-interactive) CI mode using `claude --headless`
or `claude --init-only` with appropriate tool allowlists. The Claude Code `Setup`
hook event (added in v2.1.10) fires specifically when Claude is initialized via
`claude --init` or `claude --init-only`, making it suitable for bootstrapping
TappsMCP during team onboarding pipelines. Currently TappsMCP has no
documentation or generated configuration for CI usage. This story documents
how to use TappsMCP in CI pipelines, generates GitHub Actions workflow snippets
during `tapps_init`, and explains the `enableAllProjectMcpServers` VS Code
setting that enables MCP in non-interactive contexts. The deliverables include
a generated `.github/workflows/tapps-quality.yml` snippet, inline documentation
in the generated `CLAUDE.md`, and a new `docs/ci-integration.md` guide.

### Generated GitHub Actions workflow

```yaml
# .github/workflows/tapps-quality.yml
# Generated by TappsMCP tapps_init — edit as needed
name: TappsMCP Quality Gate

on:
  pull_request:
    paths:
      - "**.py"

jobs:
  quality:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.12"

      - name: Install TappsMCP
        run: pip install tapps-mcp

      - name: Run TappsMCP quality gate
        env:
          TAPPS_MCP_PROJECT_ROOT: ${{ github.workspace }}
        run: |
          tapps-mcp validate-changed \
            --preset staging \
            --changed-files $(git diff --name-only origin/${{ github.base_ref }}...HEAD | grep '\.py$' | tr '\n' ' ')
```

### Claude Code headless usage pattern

For teams using Claude Code in CI with Setup hooks:

```bash
# Team onboarding: initialize Claude Code with TappsMCP
claude --init-only \
  --allowedTools "mcp__tapps-mcp__*" \
  --project-root /workspace

# CI quality check: run headless with tool restrictions
claude --headless \
  --allowedTools "mcp__tapps-mcp__tapps_validate_changed" \
  "Run tapps_validate_changed and exit 2 if any file fails"
```

### Setup hook configuration for CI onboarding

```json
{
  "hooks": {
    "Setup": [
      {
        "matcher": "init",
        "hooks": [
          {
            "type": "command",
            "command": ".claude/hooks/tapps-session-start.sh"
          }
        ]
      }
    ]
  }
}
```

The `Setup` hook fires only when `claude --init` or `claude --init-only` is
run, making it appropriate for CI onboarding flows that should not fire on
every session startup.

### enableAllProjectMcpServers setting

VS Code and headless Claude Code environments may not auto-load project-level
MCP servers. Add this to `.vscode/settings.json` or pass it as a CLI flag:

```json
{
  "claude.enableAllProjectMcpServers": true
}
```

Or via CLI:
```bash
claude --enable-all-project-mcp-servers --headless "..."
```

## CI Integration

TappsMCP can run in CI without an interactive session:

### Direct Python invocation (recommended for CI)

# Install TappsMCP
pip install tapps-mcp

# Validate changed files
TAPPS_MCP_PROJECT_ROOT=/workspace \
  tapps-mcp validate-changed --preset staging
```

### Claude Code headless mode
```bash
claude --headless \
  --allowedTools "mcp__tapps-mcp__tapps_validate_changed" \
  "Run tapps_validate_changed with preset=staging"
```

### VS Code / headless — enableAllProjectMcpServers
In headless or non-interactive VS Code contexts, set:
`claude.enableAllProjectMcpServers: true` in workspace settings.
```
