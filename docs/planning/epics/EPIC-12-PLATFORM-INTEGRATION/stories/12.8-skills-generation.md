# Story 12.8: Skills Generation (SKILL.md)

**Status:** Complete
**Tier:** 2
**Priority:** P1
**LOE:** 2 days
**Platform:** Claude Code + Cursor

---

## Summary

Both Claude Code and Cursor support user-defined skills â€” Markdown files with YAML
frontmatter placed in platform-specific directories that teach the agent a reusable
workflow. Skills are invoked by name (e.g., `/tapps-score`) and cause the agent to
execute the documented procedure using the MCP tools referenced in the skill body.
Unlike subagent definitions, skills do not spawn a separate agent; they inject a
prompt into the current agent's context, directing it to call specific MCP tools in
a prescribed sequence. Currently `tapps_init` does not generate any skill files,
meaning users must discover and invoke TappsMCP tools ad hoc without structured
guidance. This story adds skill generation to the `tapps_init` flow for both
platforms, producing three SKILL.md files per platform that provide ready-to-invoke
workflows for scoring, quality gating, and pre-completion validation.

## Acceptance Criteria

- `tapps_init` generates the following files for Claude Code projects:
  - `.claude/skills/tapps-score/SKILL.md`
  - `.claude/skills/tapps-gate/SKILL.md`
  - `.claude/skills/tapps-validate/SKILL.md`
- `tapps_init` generates the following files for Cursor projects (or when
  `--platform cursor` is specified):
  - `.cursor/skills/tapps-score/SKILL.md`
  - `.cursor/skills/tapps-gate/SKILL.md`
  - `.cursor/skills/tapps-validate/SKILL.md`
- All generated SKILL.md files contain valid YAML frontmatter with at minimum
  a `name` and `description` field
- Claude Code SKILL.md files include a `tools` frontmatter field listing the MCP
  tools the skill will invoke
- Cursor SKILL.md files include an `mcp_tools` frontmatter field (YAML list) naming
  the MCP tools used
- The `tapps-score` skill instructs the agent to call `tapps_score_file` on a
  target file and present a structured score report
- The `tapps-gate` skill instructs the agent to call `tapps_quality_gate` and
  report pass/fail with blocking issues listed
- The `tapps-validate` skill instructs the agent to call `tapps_validate_changed`
  and confirm all changed files meet quality thresholds before declaring work complete
- Each skill directory is created if it does not exist
  (e.g., `.claude/skills/tapps-score/` is created before writing `SKILL.md`)
- If a SKILL.md already exists at the target path, `tapps_init` skips writing it
  and logs a warning rather than overwriting user customizations
- A unit test verifies that all six SKILL.md files are created (three per platform)
  when `tapps_init` is called for each platform
- A unit test verifies the frontmatter `tools` field is present in Claude Code
  SKILL.md files and `mcp_tools` is present in Cursor SKILL.md files
- A unit test verifies the skill body references the correct MCP tool names for
  each skill
- `mypy --strict` passes with no new errors
- `ruff check` passes with no new errors

### Claude Code SKILL.md format

Claude Code skills are invoked with a `/skill-name` slash command. The frontmatter
uses a comma-separated `tools` string mirroring the subagent convention. MCP tool
references in the body use the `mcp__tapps-mcp__<tool_name>` pattern.

```markdown
---
name: tapps-score
description: Score a Python file across 7 quality categories and display a structured report.
tools: mcp__tapps-mcp__tapps_score_file, mcp__tapps-mcp__tapps_quick_check
---

Score the specified Python file using TappsMCP:

1. Call `mcp__tapps-mcp__tapps_quick_check` with the file path to get an instant score
2. If the score is below 80, call `mcp__tapps-mcp__tapps_score_file` for the full 7-category breakdown
3. Present the results in a table: category, score (0-100), top issue per category
4. Highlight any category scoring below 70 as a priority fix
5. Suggest the single highest-impact change the developer can make
```

### Cursor SKILL.md format

Cursor skills are invoked with `@skill-name`. The frontmatter uses an `mcp_tools`
YAML list instead of the comma-separated `tools` string. MCP tools in the body are
referenced by their short name (without the `mcp__tapps-mcp__` prefix).

```markdown
---
name: tapps-score
description: Score a Python file across 7 quality categories and display a structured report.
mcp_tools:
  - tapps_score_file
  - tapps_quick_check
---

Score the specified Python file using TappsMCP:

1. Call `tapps_quick_check` with the file path to get an instant score
2. If the score is below 80, call `tapps_score_file` for the full 7-category breakdown
3. Present the results in a table: category, score (0-100), top issue per category
4. Highlight any category scoring below 70 as a priority fix
5. Suggest the single highest-impact change the developer can make
```

### Skill content summary

| Skill | Key MCP tools | Purpose |
|-------|---------------|---------|
| `tapps-score` | `tapps_score_file`, `tapps_quick_check` | Score and rate a Python file across 7 categories |
| `tapps-gate` | `tapps_quality_gate` | Run quality gate check and report pass/fail with blocking issues |
| `tapps-validate` | `tapps_validate_changed` | Pre-completion validation of all changed files |

### tapps-gate skill body (both platforms, tool names vary)

The gate skill should:
1. Call `tapps_quality_gate` with the current project root (or a specified preset)
2. Display the overall pass/fail result
3. List each failing criterion with its actual vs. required value
4. If the gate fails, block the developer from declaring work complete and list
   the minimum changes required to pass

### tapps-validate skill body (both platforms, tool names vary)

The validate skill should:
1. Call `tapps_validate_changed` to get the list of changed files and their scores
2. Display each file with its score and pass/fail status against the quality gate
3. If any file fails, list it with the top issue preventing it from passing
4. Confirm explicitly when all changed files pass before the developer marks work done

### Approach

1. In `pipeline/init.py`, add a
   `_generate_skills(project_root: Path, platform: str) -> None` function that:
   a. Determines the target base directory: `.claude/skills/` for `claude`,
      `.cursor/skills/` for `cursor`
   b. For each of the three skills (`tapps-score`, `tapps-gate`, `tapps-validate`):
      i. Creates the skill subdirectory if it does not exist
      ii. Checks whether `SKILL.md` already exists at the target path
      iii. If it exists, logs a warning and skips; if not, reads the appropriate
           template from `src/tapps_mcp/prompts/skills/{platform}/{skill}/SKILL.md`
           and writes it
2. Call `_generate_skills(project_root, platform)` from the main `tapps_init`
   handler after determining the platform.
3. Include both `claude` and `cursor` skill template subdirectories in package
   data in `pyproject.toml` so they are bundled with the installed package.
