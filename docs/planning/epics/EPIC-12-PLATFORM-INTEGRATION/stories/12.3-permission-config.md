# Story 12.3: Permission Pre-Configuration

**Status:** Complete
**Tier:** 1
**Priority:** P0
**LOE:** 0.5 day
**Platform:** Claude Code

---

## Summary

Claude Code displays a permission prompt each time an MCP tool is invoked unless
that tool is explicitly allowed in the project's `.claude/settings.json`. Because
TappsMCP registers 21 tools, a session without pre-configured permissions generates
21 individual prompts — one per tool on first use. This friction discourages
adoption and breaks automated pipeline workflows. The `tapps_init` MCP tool
already bootstraps CLAUDE.md; extending its `_bootstrap_claude` method to also
write (or safely merge into) `.claude/settings.json` with a wildcard allow entry
(`"mcp__tapps-mcp__*"`) eliminates all permission prompts in one step. Existing
user settings in `.claude/settings.json` must be preserved — only the
`permissions.allow` list is updated, never overwritten wholesale.

## Acceptance Criteria

- Running `tapps_init` (or `tapps-mcp init` for Claude Code) creates
  `.claude/settings.json` if it does not exist, containing:
  ```json
  {
    "permissions": {
      "allow": [
        "mcp__tapps-mcp__*"
      ]
    }
  }
  ```
- If `.claude/settings.json` already exists and already contains
  `"mcp__tapps-mcp__*"` in `permissions.allow`, the file is not modified
  (idempotent)
- If `.claude/settings.json` already exists but does not contain the wildcard
  entry, the entry is appended to the existing `permissions.allow` list without
  touching any other keys or values in the file
- If `.claude/settings.json` exists but has no `permissions` key, a `permissions`
  block is added with the `allow` list; all other top-level keys are preserved
- The `.claude/` directory is created if it does not exist
- The `_bootstrap_claude` function in `pipeline/init.py` returns an action string
  of `'created'`, `'updated'`, or `'skipped'` for the settings file (mirroring the
  existing pattern used for CLAUDE.md)
- No other files in `.claude/` are modified by this change
- A unit test verifies each of: file created, file updated (entry appended), file
  skipped (already present), and existing-key preservation
- `mypy --strict` passes with no new errors
- `ruff check` passes with no new errors

## Implementation Notes

### Files to change

| File | Change |
|------|---------|
| `src/tapps_mcp/pipeline/init.py` | Add `_bootstrap_claude_settings()` helper; call it from the Claude bootstrap path |

### Approach

1. Add a new private helper `_bootstrap_claude_settings(project_root: Path) -> str`
   in `pipeline/init.py`. This function:
   - Determines the settings file path: `project_root / ".claude" / "settings.json"`
   - Creates the `.claude/` directory if missing (`mkdir(parents=True, exist_ok=True)`)
   - If the file does not exist, writes the minimal JSON shown in the acceptance
     criteria and returns `'created'`
   - If the file exists, parses it with `json.loads`, navigates to (or creates)
     `permissions.allow`, checks whether `"mcp__tapps-mcp__*"` is already present:
     - If present: returns `'skipped'`
     - If absent: appends the entry and writes the file back with `json.dumps(...,
       indent=2)`, returns `'updated'`

2. Call `_bootstrap_claude_settings(project_root)` from within `_bootstrap_claude`
   (or from the caller that invokes `_bootstrap_claude`) so it runs as part of the
   Claude Code bootstrap step.

3. Use `json.dumps(config, indent=2)` for consistent formatting. Terminate the file
   with a single newline (`"\n"`) after the closing brace.

4. The wildcard entry to add/check for is exactly `"mcp__tapps-mcp__*"`. Do not
   expand it to the full 21-item list — the wildcard is simpler and forward-compatible
   with any future tools added to TappsMCP.

5. All file I/O must use `encoding="utf-8"` and `Path.read_text` / `Path.write_text`.

### Example output — new file

```json
{
  "permissions": {
    "allow": [
      "mcp__tapps-mcp__*"
    ]
  }
}
```

### Example output — merge into existing file

Existing:
```json
{
  "theme": "dark",
  "permissions": {
    "allow": [
      "Bash(*)"
    ]
  }
}
```

After merge:
```json
{
  "theme": "dark",
  "permissions": {
    "allow": [
      "Bash(*)",
      "mcp__tapps-mcp__*"
    ]
  }
}
```

## Testing Requirements

### Unit tests — `tests/unit/test_init_bootstrap.py` (existing file, add cases)

- **Created**: Call `_bootstrap_claude_settings` with a temp directory containing
  no `.claude/settings.json`. Assert the function returns `'created'`, the file
  exists, and `json.loads(file.read_text())["permissions"]["allow"]` contains
  `"mcp__tapps-mcp__*"`.

- **Skipped**: Call `_bootstrap_claude_settings` when the file already contains the
  wildcard entry. Assert the function returns `'skipped'` and the file is unchanged.

- **Updated — append to allow list**: Provide a settings.json with an existing
  `permissions.allow` list that does not contain the wildcard. Assert the function
  returns `'updated'`, the wildcard is added, and the original entry is still
  present.

- **Updated — no permissions key**: Provide a settings.json with only non-permissions
  keys (e.g., `{"theme": "dark"}`). Assert the function returns `'updated'`, the
  `permissions.allow` list is created with the wildcard, and `"theme"` is still
  present.

- **Directory creation**: Call `_bootstrap_claude_settings` with a temp directory
  where `.claude/` does not exist. Assert the directory is created and the file is
  written successfully.
