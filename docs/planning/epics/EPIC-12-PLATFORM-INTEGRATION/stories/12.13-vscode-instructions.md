# Story 12.13: VS Code / Copilot Instructions

**Status:** Complete
**Tier:** 4
**Priority:** P3
**LOE:** 0.5 day
**Platform:** VS Code

---

## Summary

VS Code Copilot Chat reads `.github/copilot-instructions.md` for project-level
guidance that is automatically injected into every Copilot conversation. Currently
TappsMCP's `tapps_init` MCP tool and `tapps-mcp init` CLI generate configuration
for Claude Code and Cursor but produce nothing for VS Code users. This story adds
generation of `.github/copilot-instructions.md` to the `tapps_init` flow, giving
VS Code Copilot users a working reference to the TappsMCP tools available in their
session, the recommended workflow, and the key tools to use at each stage. The file
requires no frontmatter â€” plain markdown is sufficient. This is a low-effort,
high-value addition that closes the VS Code gap identified in the platform
comparison research.

## Acceptance Criteria

- `tapps_init` generates `.github/copilot-instructions.md` in the target project
  directory
- `tapps-mcp init` also generates `.github/copilot-instructions.md` when VS Code
  is the selected platform (or when generating for all platforms)
- The generated file is valid markdown with no YAML frontmatter
- The file includes a top-level heading: `# TappsMCP Quality Tools`
- The file includes a "Key Tools" section listing at minimum:
  - `tapps_session_start` with a one-line description
  - `tapps_quick_check` with a one-line description
  - `tapps_quality_gate` with a one-line description
  - `tapps_validate_changed` with a one-line description
- The file includes a "Workflow" section with numbered steps covering:
  1. Starting a session with `tapps_session_start`
  2. Running `tapps_quick_check` after editing Python files
  3. Running `tapps_validate_changed` before declaring work complete
- The file includes a brief introductory sentence explaining that the project
  uses TappsMCP for code quality analysis
- The file references the `.vscode/mcp.json` configuration file for MCP server
  setup
- `tapps_init` creates the `.github/` directory if it does not already exist
- The generation is idempotent: re-running `tapps_init` overwrites the file with
  the same content
- A unit test verifies the file is created with expected content sections
- A unit test verifies the file contains references to all four key tool names
- `mypy --strict` passes with no new errors
- `ruff check` passes with no new errors

### Generated file content

```markdown
# TappsMCP Quality Tools

This project uses TappsMCP for code quality analysis. When TappsMCP is
available as an MCP server (configured in `.vscode/mcp.json`), use the
following tools to maintain code quality throughout development.

## Key Tools

- `tapps_session_start` â€” Initialize a TappsMCP session at the start of
  each work session. Call this first.
- `tapps_quick_check` â€” Run a quick quality check on a single file after
  editing. Returns score and top issues.
- `tapps_quality_gate` â€” Run a pass/fail quality gate against a configurable
  preset (development, staging, or production).
- `tapps_validate_changed` â€” Validate all changed files against the quality
  gate. Call this before declaring work complete.
- `tapps_consult_expert` â€” Consult a domain expert (security, performance,
  architecture, testing, and more) for guidance.
- `tapps_score_file` â€” Get a detailed 7-category quality score for any file.

## Workflow

1. Start a session: call `tapps_session_start`
2. After editing Python files: call `tapps_quick_check` on changed files
3. Before creating a PR or declaring work complete: call `tapps_validate_changed`
4. For domain-specific guidance: call `tapps_consult_expert` with the
   relevant domain

## Quality Scoring Categories

TappsMCP scores code across 7 categories (0-100 each):
correctness, security, maintainability, performance, documentation,
testing, and style.
```

### Approach

1. Add a `prompts/vscode/copilot-instructions.md` template file.
2. Add a `generate_copilot_instructions(output_dir: Path) -> Path` function
   to `setup_generator.py` that writes the template to
   `output_dir / ".github" / "copilot-instructions.md"`, creating the
   `.github/` directory if needed.
3. Call `generate_copilot_instructions` from both `pipeline/init.py`
   (for the `tapps_init` MCP tool) and the CLI init path.
4. The function should be unconditional â€” generate the file for all platforms,
   not only when VS Code is explicitly selected, since projects may use
   multiple editors.

### VS Code MCP configuration note

VS Code uses `.vscode/mcp.json` (not `.mcp.json`). The copilot instructions
should reference this path explicitly. Story 12.3 or the existing init flow
should also generate `.vscode/mcp.json`; if it does not, note it as a
prerequisite dependency.
