# Epic 14: Diagram Generation Quality

**Status:** Complete
**Priority:** P1
**Estimated LOE:** ~1–1.5 weeks
**Dependencies:** DocsMCP Epic 6 (Diagram Generation)
**Blocks:** None

---

## Goal

Fix DocsMCP diagram generation so it scans actual project source directories instead of pip/venv internals, produces package-scoped diagrams for monorepos, and warns when results appear degraded.

## Current State

All 7 diagrams generated by DocsMCP against TappsMCP contain only pip/setuptools internals (`_TrivialRe`, `DistutilsMetaFinder`, `shim`). Root cause: `_collect_classes()` in `diagrams.py` uses `project_root.rglob("*.py")` without source directory auto-detection, scanning `.venv/`, `dist/`, and other non-source directories.

The fix pattern already exists in two sibling analyzers:
- `dependency.py._resolve_source_dirs()` — detects `src/`, `packages/*/src/`, flat layouts
- `module_map.py._resolve_source_dirs()` — same pattern, independently implemented

## Acceptance Criteria

- [ ] Diagrams scan only detected source directories, not `project_root` recursively
- [ ] Shared `_SKIP_DIRS` constant used across all generators/analyzers
- [ ] Monorepo projects get per-package diagrams with package-prefixed titles
- [ ] Quality warnings emitted when <3 classes/modules found
- [ ] All existing DocsMCP tests pass; new tests cover monorepo, single-src, flat layouts

## Stories

### 14.1 — Source Directory Auto-Detection for Diagrams

**Priority:** Critical | **Points:** 5

Extract or reuse `_resolve_source_dirs()` and wire into all collection functions in `diagrams.py`.

**Source Files:**
- `packages/docs-mcp/src/docs_mcp/generators/diagrams.py` — wire `_resolve_source_dirs()`
- `packages/docs-mcp/src/docs_mcp/analyzers/dependency.py` — reference implementation
- `packages/docs-mcp/src/docs_mcp/analyzers/module_map.py` — reference implementation

**Tasks:**
- Extract `_resolve_source_dirs()` into a shared utility or import from existing module
- Update `_collect_classes()` to iterate over resolved source dirs instead of `project_root.rglob()`
- Update `_collect_dependencies()` and `_collect_modules()` similarly
- Add tests: monorepo (`packages/*/src/`), single `src/` layout, flat layout with `__init__.py`

**Definition of Done:** Running DocsMCP diagrams against TappsMCP produces classes from `tapps_core`, `tapps_mcp`, and `docs_mcp` — not pip internals.

### 14.2 — Shared SKIP_DIRS Constant

**Priority:** High | **Points:** 2

Unify `_SKIP_DIRS` across all modules that walk Python files.

**Source Files:**
- `packages/docs-mcp/src/docs_mcp/extractors/base.py` or new `constants.py` — define constant
- `packages/docs-mcp/src/docs_mcp/generators/diagrams.py` — use shared constant
- `packages/docs-mcp/src/docs_mcp/generators/api_docs.py` — use shared constant
- `packages/docs-mcp/src/docs_mcp/analyzers/module_map.py` — use shared constant
- `packages/docs-mcp/src/docs_mcp/analyzers/dependency.py` — use shared constant

**Tasks:**
- Create `SKIP_DIRS` set with: `.venv`, `venv`, `__pycache__`, `.git`, `node_modules`, `.tox`, `dist`, `build`, `.eggs`, `.mypy_cache`, `.ruff_cache`, `.pytest_cache`, `site-packages`
- Replace per-module `_SKIP_DIRS` with import from shared location
- Verify no behavior change via existing tests

**Definition of Done:** Single source of truth for skip directories; all modules import from it.

### 14.3 — Package-Scoped Diagram Generation

**Priority:** Medium | **Points:** 2

Generate per-package diagrams for monorepo projects.

**Source Files:**
- `packages/docs-mcp/src/docs_mcp/generators/diagrams.py` — per-package logic

**Tasks:**
- When `_resolve_source_dirs()` returns multiple dirs, generate one diagram per source dir
- Prefix diagram titles with package name (e.g., "tapps-core Class Hierarchy")
- Add `scope` parameter: `"package"` (per-package) or `"project"` (combined)
- Return list of diagrams in result when multiple packages detected

**Definition of Done:** TappsMCP dogfood produces separate diagrams for tapps-core, tapps-mcp, and docs-mcp.

### 14.4 — Diagram Quality Warnings

**Priority:** Low | **Points:** 2

Detect and report when diagram generation produces suspicious results.

**Source Files:**
- `packages/docs-mcp/src/docs_mcp/generators/diagrams.py` — warning logic

**Tasks:**
- After collection, check if <3 classes/modules found; if so, add `degraded: true` to result
- Include `scanned_dirs` and `skipped_count` in result metadata
- Log warning via structlog when degraded results detected

**Definition of Done:** Degraded results are flagged; metadata shows what was scanned.
