# Epic 48: MCP Host Visibility & Agent Fallbacks

**Status:** Complete
**Priority:** P1
**Estimated LOE:** ~1 week
**Dependencies:** None (docs + content)
**Blocks:** None
**Source:** Consuming project feedback — `OpenClawAgents/docs/tapps-mcp-feedback.md` (2026-03-05)

---

## Goal

Ensure consuming projects and AI agents can succeed when TappsMCP is configured but not yet visible to the agent’s MCP layer, and document how to verify and fix MCP server availability. When the host (Cursor/Claude Code/VS Code) has tapps-mcp in config files but the agent only sees a subset of servers (e.g. `user-context7`, `user-GitHub`, `cursor-ide-browser`), skills and rules that say “call `tapps_init`” or “call `tapps_session_start` first” cannot be followed. This epic adds documentation and in-skill fallbacks so agents and users have a clear path to bootstrap and verify.

## Problem Statement

Feedback from OpenClawAgents (session 2026-03-05) reported:

1. **MCP server not available to the agent (blocking):** The project had tapps-mcp in `.cursor/mcp.json`, `.mcp.json`, and `.vscode/mcp.json`, but the agent’s `call_mcp_tool(server="tapps-mcp", ...)` failed with “MCP server does not exist: tapps-mcp. Available servers: user-context7, user-GitHub, cursor-ide-browser.” So the server was configured for the IDE but not exposed to the agent in that session.
2. **tapps-init skill assumes MCP is always callable:** The tapps-init skill instructs “Call `tapps_init`” without a fallback. When the server is missing from the agent’s available servers, step 1 is impossible and there is no in-skill guidance to use the CLI.

### Impact

| Issue | Severity | Workaround used in feedback |
|-------|----------|------------------------------|
| tapps-mcp server not in agent’s available MCP servers | High | Use CLI: `tapps-mcp upgrade`, `tapps-mcp doctor` |
| tapps-init skill has no CLI fallback | Medium | Same CLI commands as above |

## Stories

### Story 48.1: Document MCP server visibility and verification

**LOE:** S (~2–3 hours)
**Files:** `AGENTS.md`, `docs/` (onboarding or troubleshooting), optionally `README.md`

Document that tapps-mcp must be **enabled and listed as an available MCP server** in the host (Cursor / Claude Code / VS Code) for agent-invoked tools to work. Add:

- **Requirement:** MCP tools (e.g. `tapps_init`, `tapps_session_start`, `tapps_quick_check`) are only callable when the tapps-mcp server appears in the host’s “available servers” list for the current session.
- **Verification:** How to confirm the server is available (e.g. Cursor: check MCP panel/settings; Claude Code: check MCP config and restart if needed).
- **CLI fallback:** If MCP tools are unavailable, run from the project root: `tapps-mcp upgrade --force --host auto` (or appropriate host), then `tapps-mcp doctor` to bootstrap and verify. Optionally mention Cursor-specific steps (e.g. enabling the server in settings or restarting) so users get tapps-mcp into the available servers list.

**Acceptance Criteria:**

- AGENTS.md (or linked doc) states that agent tools require the tapps-mcp server to be available in the host.
- Docs explain how to verify the server is available and what to do if it is not (CLI fallback + host-specific hints where applicable).
- No code changes required; documentation only.

---

### Story 48.2: Add CLI fallback to tapps-init skill and tool reference

**LOE:** S (~1–2 hours)
**Files:** `.cursor/skills/tapps-init/SKILL.md` (or equivalent), AGENTS.md / tapps-tool-reference skill content

Add a short fallback to the tapps-init flow for when `tapps_init` is not callable (e.g. server not in available MCP servers):

- In the **tapps-init skill:** After “Call `tapps_init`”, add: “If `tapps_init` is not available (e.g. server not in available MCP servers), run from the project root: `tapps-mcp upgrade --force --host auto` then `tapps-mcp doctor` to bootstrap and verify.”
- In **AGENTS.md** and/or **tapps-tool-reference** (skill or docs): Add the same fallback where `tapps_init` is described, so agents and users see it when looking up init behavior.

**Acceptance Criteria:**

- Tapps-init skill (and any generated copy in consuming projects) includes the CLI fallback sentence.
- AGENTS.md / tool-reference content for `tapps_init` includes the same fallback.
- Wording is consistent and points to `tapps-mcp upgrade` + `tapps-mcp doctor` as the bootstrap/verify path.

## Out of Scope

- Changing how Cursor/Claude Code/VS Code expose MCP servers to the agent (host behavior).
- Adding new MCP tools or CLI flags; this epic is documentation and content only (plus any template/skill text that gets generated by tapps-mcp).

## Success Criteria

- Users and agents have a documented way to verify tapps-mcp is available and what to do when it is not.
- When the agent cannot call `tapps_init`, the tapps-init skill and tool reference direct them to the CLI fallback so bootstrap and verification can still complete.
