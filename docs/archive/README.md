# Archived Documentation

Regeneratable or supplementary docs kept out of the main tree to reduce noise for LLMs and developers.

## Contents

| Directory | Description |
|---|---|
| `api/` | Auto-generated API reference (regenerate with `docs_generate_api`) |
| `diagrams/` | Auto-generated architecture diagrams (regenerate with `docs_generate_diagram`) |
| `reference/` | Supplementary reference (migration guides, composition, consumer requirements) |
| `smoke/` | DocsMCP smoke-test fixtures |

Planning epics and historical reviews were removed — **Linear is the backlog history**; code and ADRs are the implementation truth.

## Where to look instead

- Architecture: `docs/ARCHITECTURE.md`
- Config: `docs/CONFIG_REFERENCE.md`
- Setup: `docs/ONBOARDING.md`, `docs/TROUBLESHOOTING.md`
- Tool surface: `AGENTS.md`, `docs/architecture/tool-budget.md`
- Decisions: `docs/adr/`
