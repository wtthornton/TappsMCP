When a task asks to modify Ralph's behavior — coordinator brief/debrief flow, agent tool allowlists, on-stop/on-subagent-done hooks, ralph-workflow SKILL — the edit belongs in the upstream `ralph-claude-code` repo, NOT in tapps-mcp.

**Why:** `.ralph/hooks/protect-ralph-files.sh` (wired as a PreToolUse hook in `.claude/settings.json`) hard-blocks Edit/Write on `.claude/agents/*`, `.claude/hooks/*`, `.claude/settings*.json`, and `.claude/commands/*`. It exits 2 with `BLOCKED: Cannot modify Claude Code agent/hook config: <path>`. The block is by design (TAP-623): under `bypassPermissions` mode a single misread fix_plan.md line could otherwise disable every guardrail permanently. The agent files in `.claude/agents/ralph*.md` and the skill at `.claude/skills/ralph-workflow/SKILL.md` are deployed copies, regenerated by `ralph-upgrade` from `ralph-claude-code`. There is no template source for them under `packages/tapps-mcp/src/tapps_mcp/prompts/` (verified: only `agents_template*.md`, `platform_*.md`, pipeline-stage prompts, and `karpathy_guidelines.md` live there — no `ralph*.md` or `coordinator*`).

**How to apply:** If a task brief in tapps-mcp says "edit `.claude/agents/ralph.md`" or "wire X into Ralph's coordinator/skill/hooks", stop and hand back a prompt scoped to the `ralph-claude-code` repo. Don't try to bypass the protect hook (no `TAPPS_PROTECT_BYPASS=` env var exists; the hook unconditionally exits 2). The `feedback_template_sync` rule still applies in the OTHER direction: if a tapps-mcp template under `packages/tapps-mcp/src/tapps_mcp/prompts/` IS the source for a deployed file under `.claude/`, edit the template — but ralph-* files are not in that set.

Tasks like TAP-1712 story 0.2 ("wire `tapps_memory(action=recall_many|related|rate)` into Ralph's cold-start path") are in scope for `ralph-claude-code`, not tapps-mcp. The tapps-mcp side of TAP-1712 would only be: (a) ensuring the `tapps_memory` actions are exposed (already done by TAP-1628 / commits 51a71e6, f3840cb, b4e97c5, 679841b — see `packages/tapps-mcp/src/tapps_mcp/server_memory_tools.py:50-148`), or (b) docs/AGENTS.md surfacing of the new actions (already done by commit 7454018).

See [[feedback-repo-boundaries]] for the broader rule.
